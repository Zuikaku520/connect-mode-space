# Theory Framework

## Core Claim
System output is determined not only by input and node properties, but also by connection mode.

## Minimal Formula
Behavior(t) = F(Input(t), Nodes, Weights, ConnectionMode)

## Distinction
Traditional emphasis:
- Node function
- Weight value

This project adds:
- Connection mode as an independent dynamical dimension

## Structured Mode Space Hypothesis

Current experiments suggest that connection mode space is not homogeneous.

Instead, it appears to contain:
- near-baseline regions
- mid-shift regions
- far-shift regions
- localized synergy windows

This means that connection mode combinations do not uniformly generate new behavior.
Some remain close to parent modes, while others enter distinct high-synergy regions.

## Working Definitions

### Common Line
A simplified assumption in which node-to-node connections are treated as uniform weighted transmission edges.

### Connection Mode
A structured connection implementation that can alter propagation dynamics, including delay, recurrence, inhibition, gating, or higher-order combinations.

### Mode Space
The parameterized space formed by possible connection modes and their combinations.

### Synergy Window
A localized parameter region in mode space where combined modes produce new dynamical behavior beyond what parent modes individually explain.

A local region in parameter space where combined modes consistently generate stronger dynamical deviation than parent modes alone.

## Working Hypothesis
Current AI systems may largely optimize within a “connection baseline / common line” regime, while more advanced cognitive architectures may require access to higher-order connection mode regions.
